## 二、需求背景

### 2.1 要解决的问题

中小型私立医疗机构（口腔诊所、体检中心、健康管理机构等）普遍面临以下运营痛点：

**痛点一：前台重复咨询压力大** 患者每天通过微信、电话、公众号咨询大量重复性问题（如何挂号、是否需要空腹、检查项目价格等）。人工前台响应速度慢、工作时间有限，且成本高（月薪 4500–6000 元）。

**痛点二：无法 24 小时响应** 患者咨询高峰往往在非工作时间，人工前台下班后无人响应，导致患者流失。

**痛点三：没有数据沉淀** 患者问过哪些问题、哪类问题最集中、哪个科室咨询转化率最低，完全没有数据记录，无法指导运营决策。

**痛点四：预约管理粗放** 大量诊所仍依赖电话或微信群手动管理预约，容易漏单、重复预约，管理效率低。

### 2.2 用户调研结论

**调研方式：** 公开平台数据分析（大众点评、小红书用户评价）+ 行业公开报告参考

**关键结论：**

- 大众点评上，"电话打不通"、"没人回消息"是私立诊所差评的高频词
- 小红书上大量用户抱怨"问个价格问了半天"、"预约流程太麻烦"
- 中国现有民营医疗机构超过 68 万家（2023年数据），其中中小型机构占绝大多数，普遍缺乏技术团队
- 同类 SaaS 产品（医疗行业 CRM、在线问诊平台）月均订阅价格在 1000–5000 元区间，市场对付费服务接受度已初步形成

### 2.3 竞品分析结论

| 竞品                          | 主要能力                                 | 不足之处                                         | 关键结论               |
| ----------------------------- | ---------------------------------------- | ------------------------------------------------ | ---------------------- |
| 微脉                          | 面向大型医院的患者服务平台，含问诊、预约 | 主要服务公立医院，中小私立机构无法负担其部署成本 | 市场空白在中小私立机构 |
| 健康160                       | 预约挂号平台                             | 纯预约工具，无 AI 问答能力，不支持私有化知识库   | 缺乏 AI 能力是普遍缺口 |
| 通用 AI 客服（如小满、Udesk） | 通用行业 AI 客服                         | 非医疗垂直，知识库需大量定制，上手成本高         | 医疗垂直场景存在机会   |

**结论：** 当前市场缺乏专门面向中小私立医疗机构、开箱即用的 AI 运营服务平台。医云问定位于此细分市场。

## 三、需求目标

**需求目标：** 为中小型私立医疗机构提供一站式 AI 运营服务平台，涵盖 AI 智能客服、预约挂号管理、知识库文章管理三大核心功能，帮助机构降低人工咨询成本、提升患者服务体验。

**衡量标准：**

| 指标                  | 目标值                                 |
| --------------------- | -------------------------------------- |
| AI 客服问题响应准确率 | 基于知识库内容的问题，准确回答率 ≥ 85% |
| 系统并发支持          | 单机构同时支持 ≥ 50 路对话             |
| 预约流程完成时长      | 患者从发起预约到预约确认 ≤ 2 分钟      |
| 机构入驻配置时长      | 新机构从注册到 AI 客服上线 ≤ 30 分钟   |

## 五、功能详细说明

### 5.1 产品流程图

**核心流程：患者使用 AI 客服并预约**

```Plain
患者访问页面
    ↓
输入问题（如"洗牙多少钱"）
    ↓
系统检索机构知识库（RAG）
    ↓
AI 生成回答并返回（流式输出）
    ↓
患者决定预约 → 点击"立即预约"
    ↓
选择科室 → 选择时间 → 填写姓名/手机号
    ↓
提交预约
    ↓
机构后台收到预约通知 → 管理员确认
    ↓
患者预约成功
```

**机构入驻流程：**

```Plain
机构注册账号 → 填写机构信息
    ↓
上传知识库文档（科室介绍、价格表、常见问题等）
    ↓
系统解析文档并建立向量索引
    ↓
AI 客服上线，可对外服务
    ↓
配置号源（科室、医生、可预约时段）
    ↓
发布健康科普文章（可选）
```

### 5.2 功能点说明

#### 模块一：AI 智能客服

| 功能         | 功能详细说明                                                 |
| ------------ | ------------------------------------------------------------ |
| 知识库问答   | 患者输入问题后，系统通过 RAG 技术检索机构上传的知识库内容，结合大语言模型生成准确回答。回答仅基于知识库内容，避免 AI 编造信息 |
| 流式输出     | AI 回答逐字输出（类似 ChatGPT 效果），提升交互体验，减少等待感 |
| 超出范围提示 | 当问题超出知识库范围时，AI 明确告知"此问题需联系人工客服"，不胡乱回答 |
| 对话历史     | 同一会话内保留上下文，支持多轮对话                           |

#### 模块二：预约挂号管理

| 功能       | 功能详细说明                                                 |
| ---------- | ------------------------------------------------------------ |
| 号源配置   | 机构管理员在后台配置科室列表、出诊医生、可预约时间段及每时段号源数量 |
| 在线预约   | 患者选择科室 → 选择日期和时间段 → 填写姓名、手机号 → 提交预约，系统自动扣减号源 |
| 预约管理   | 管理员查看所有预约记录，可确认或取消预约，取消后号源自动释放 |
| 防重复预约 | 同一手机号在同一时段只能预约一次，通过 Redis 加锁实现        |

#### 模块三：知识库文章管理

| 功能     | 功能详细说明                                                 |
| -------- | ------------------------------------------------------------ |
| 文章发布 | 管理员在富文本编辑器中撰写健康科普文章，支持图片上传（存储至 OSS） |
| 文章管理 | 支持文章的上线/下线、编辑、删除操作                          |
| 患者浏览 | 患者端展示已上线文章列表，可点击查看全文                     |
| 分类标签 | 文章支持打标签（如"口腔健康"、"体检须知"），便于分类展示     |

## 六、非功能需求

| 类型         | 要求                                                         |
| ------------ | ------------------------------------------------------------ |
| 性能         | API 接口响应时间 P99 ≤ 500ms（AI 对话接口除外）              |
| 安全         | JWT Token 认证，Token 过期时间 2 小时；用户密码 bcrypt 加密存储；接口限流防刷（Redis 实现） |
| 可用性       | 核心服务（API + 数据库）保证可用，通过 Docker Compose 一键部署 |
| 数据隔离     | 多租户架构，每家机构数据严格隔离，不同机构间数据不可互访     |
| 缓存穿透防护 | 使用布隆过滤器（Bloom Filter）防止无效机构 ID 查询打穿数据库 |
| 异步解耦     | 对话记录写入 Elasticsearch 通过 RabbitMQ 消息队列异步处理，不影响主流程响应速度 |

## 七、数据需求

| 数据名称     | 数据口径                             | 用途                             |
| ------------ | ------------------------------------ | -------------------------------- |
| 日对话量     | 每日发起对话的会话数                 | 反映平台活跃度                   |
| 问题类型分布 | 对话内容关键词聚类统计 Top10         | 帮助机构了解患者最关心的问题     |
| 预约转化率   | 发起预约人数 / 对话人数              | 衡量 AI 客服对预约业务的拉动效果 |
| 知识库命中率 | AI 基于知识库回答的次数 / 总回答次数 | 评估知识库内容质量               |

## 八、项目规划

### 技术架构

| 层次     | 技术选型                         | 用途                                 |
| -------- | -------------------------------- | ------------------------------------ |
| 前端     | Vue3 + TypeScript + Element Plus | 管理后台 + 患者端页面                |
| 后端     | FastAPI + SQLAlchemy ORM         | RESTful API 服务                     |
| 数据库   | MySQL 8.0                        | 业务主数据存储                       |
| 缓存     | Redis 7.0                        | 会话缓存、限流、布隆过滤器、分布式锁 |
| 搜索引擎 | Elasticsearch 8.x                | 对话记录全文检索                     |
| 消息队列 | RabbitMQ                         | 异步写入 ES、解耦主流程              |
| 对象存储 | OSS（阿里云 / MinIO）            | 文档、图片文件存储                   |
| AI 引擎  | Dify（RAG + Agent 编排）         | 知识库检索 + AI 回答生成             |
| 认证     | JWT（JSON Web Token）            | 多租户身份认证与权限控制             |
| 部署     | Docker + Docker Compose          | 一键启动所有服务                     |
